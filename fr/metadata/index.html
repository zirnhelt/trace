TRACE / Norme de métadonnées

Implémenter
les métadonnées TRACE

Les évaluations TRACE sont conçues pour être lisibles par les machines. Les intégrer dans votre contenu rend la responsabilité trouvable, indexable et vérifiable — sans que quiconque doive visiter manuellement la fiche d'évaluation. Trois formats existent pour différents contextes de publication.

Choisir un format
Format 01
JSON-LD
application/ld+json
Données liées structurées intégrées dans le <head> HTML ou retournées depuis une API. Lisibles par les moteurs de recherche, les agrégateurs et tout outil compatible JSON. Le format par défaut pour le contenu publié sur le web.
Idéal pour
  • Pages web et articles
  • Réponses d'API REST
  • Plateformes avec accès CMS
  • Données de responsabilité visibles par les moteurs de recherche
Voir la spécification →
Format 02
XML
espace de noms trace:
Une extension d'espace de noms XML pour RSS, Atom et d'autres formats de flux. Ajoutez un bloc trace: aux éléments de canal ou d'élément de votre flux. Lisible nativement par les applications de podcast et les agrégateurs de flux qui supportent les extensions d'espace de noms.
Idéal pour
  • Flux RSS de podcasts
  • Flux Atom et d'actualités
  • Formats de documents XML
  • Écosystèmes Podcasting 2.0
Voir la spécification →
Format 03
C2PA
assertion trace.assessment
Un type d'assertion TRACE dans la norme de la Coalition pour la provenance et l'authenticité du contenu. Signé cryptographiquement et intégré directement dans les fichiers médias — survit au téléchargement, à l'hébergement et à la migration de plateforme. Le choix de durabilité.
Idéal pour
  • Images, vidéos et fichiers audio
  • PDFs et documents
  • Médias générés par l'IA
  • Contenu d'archivage à long terme
Voir la spécification →

Les formats sont complémentaires, pas exclusifs. Un épisode de podcast pourrait porter des métadonnées TRACE dans les trois formats simultanément : JSON-LD sur la page de l'épisode, un bloc trace: dans le flux RSS, et une assertion C2PA intégrée dans le fichier audio. Chacun sert un lecteur différent dans un contexte différent.
Ce que tous les formats contiennent

Quel que soit le format, chaque objet de métadonnées TRACE encode les mêmes informations de base. Les noms de champs diffèrent selon le format; le modèle de données est identique.
