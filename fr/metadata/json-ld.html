Métadonnées / JSON-LD

JSON-LD

Intégrez les données d'évaluation TRACE en JSON-LD structuré dans votre HTML. Lisible par les moteurs de recherche, les plateformes CMS et tout processeur JSON-LD. La méthode la moins contraignante pour ajouter des métadonnées TRACE au contenu publié sur le web.

@context: https://tracestandard.org/schema/v1
@type: TraceAssessment
Référence des propriétés
Champ Type Requis Description
@context string Requis Doit être exactement "https://tracestandard.org/schema/v1"
@type string Requis Doit être exactement "TraceAssessment"
project string Requis Le nom du projet de contenu évalué
track "A" | "B" Requis Voie A (perspective communautaire interne) ou Voie B (externe / institutionnel)
verdict "Exemplaire" | "Solide" | "En développement" | "Non conforme" Requis Le verdict d'évaluation
totalScore integer Requis Points totaux obtenus
maxScore integer Requis Points maximum possibles (21 pour la Voie A, 38 pour la Voie B)
assessedDate Date ISO 8601 Requis Date à laquelle l'évaluation a été complétée (AAAA-MM-JJ)
assessorName string Optionnel Nom de la personne ou de l'organisation qui a conduit l'évaluation
assessorURL URL Optionnel URL du profil ou de la page d'organisation de l'évaluateur
scores object Optionnel Détail des scores par catégorie (voir Objet scores ci-dessous)
improvementPlan string Optionnel URL ou brève description du plan d'amélioration (requis pour le verdict En développement)
nextReviewDate Date ISO 8601 Optionnel Date prévue de la prochaine évaluation
Objet scores

La propriété scores est un objet avec des identifiants de catégorie comme clés. Chaque valeur est un objet avec les propriétés score et max.

"scores": {
  "T": { "score": 3, "max": 5 },
  "R": { "score": 3, "max": 4 },
  ...
}

Exemples
Exemple 1 — Minimal (Voie A)
<script type="application/ld+json">
{
  "@context": "https://tracestandard.org/schema/v1",
  "@type": "TraceAssessment",
  "project": "Cariboo Signals — L'épisode sur l'élevage",
  "track": "A",
  "verdict": "Solide",
  "totalScore": 15,
  "maxScore": 21,
  "assessedDate": "2026-05-28"
}
</script>
Exemple 2 — Complet avec scores (Voie A)

  "@type": "TraceAssessment",

  "verdict": "Solide",
  "totalScore": 15,

    "T": { "score": 4, "max": 5 },
    "R": { "score": 3, "max": 4 },

Modèle d'intégration HTML

Placez la balise script dans le <head> de la page HTML qui héberge le contenu. Une balise par page — si une page contient plusieurs éléments évalués, intégrez l'évaluation de l'élément principal.

<!DOCTYPE html>
<html lang="fr-CA">
<head>
  <meta charset="UTF-8" />
  <title>Titre de l'article</title>

  <!-- Métadonnées d'évaluation TRACE -->
  <script type="application/ld+json">
  {
    "@context": "https://tracestandard.org/schema/v1",
    "@type": "TraceAssessment",
    "project": "Nom de votre projet",
    "track": "A",
    "verdict": "Solide",
    "totalScore": 15,
    "maxScore": 21,
    "assessedDate": "2026-05-28"
  }
  </script>
</head>
<body>
  <!-- contenu de la page -->
</body>
</html>
Plusieurs blocs JSON-LD sur une seule page sont valides. Placez les données TRACE dans leur propre bloc plutôt que de les fusionner avec d'autres types schema.org.
Outils de validation

Utilisez ces outils pour vérifier que votre JSON-LD est bien formé avant de le publier.

Google Rich Results Test
search.google.com/test/rich-results

Valide les données structurées et prévisualise la façon dont Google les analyse.

JSON-LD Playground
json-ld.org/playground

Analyseur et validateur JSON-LD interactif.

Schema.org Validator
validator.schema.org

Vérifie les données structurées compatibles avec schema.org.

Validateur TRACE À venir
tracestandard.org/validate

Validera les règles de schéma et les seuils spécifiques à TRACE.
